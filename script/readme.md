## 一、任务背景（CCKS 2021）
- **主办**：CCKS 2021大会
- **两个子任务**：
  1. 中文地址要素解析
  2. 地址文本相关性 ← 本任务
- **数据来源**：公开地址信息（如黄页网站）
- **目的**：推动中文地址处理领域发展

## 二、任务定义
给定：
- 一个地址 **query**
- 若干个候选地址 **candidate**

输出每个 `(query, candidate)` 的匹配程度：
- **完全匹配**
- **部分匹配**
- **不匹配**

## 三、地址文本的常见形式
1. 规范地址（含四级行政区划 + 路名路号 + POI）
2. 要素缺省地址（只有路名+路号，或只有POI）
3. 非规范/口语化地址（如：*阿里西溪园区东门旁亲橙里*）

## 四、核心挑战
- 同一地址多种写法，无固定改写表
- 需要结合省市区等限制条件
- 不同城市地址规范差异大

## 五、匹配判定规则（重点）

### ✅ 完全匹配
- 匹配到**楼栋号**
- 若query只到POI，则匹配到POI即算完全匹配
- **特例**：路名+路号+POI 匹配到 路名+路号 → 完全匹配

### ⚠️ 部分匹配（常见情况）
| Query | Candidate | 判定 |
|-------|-----------|------|
| 楼栋号 | 只到POI | 部分匹配 |
| 村+村组 | 只到村 | 部分匹配 |
| POI+子POI | 只到POI | 部分匹配 |
| 村+门牌号+POI | 只到门牌号 | 部分匹配 |
| 村 | 村+POI（匹配到村） | 部分匹配 |
| 村+路+路号 | 只到村或路 | 部分匹配 |

### ❌ 不匹配
- 地址要素基本不重合
- 例：村+POI 只匹配到村（不匹配）

## 六、数据文件
| 文件 | 用途 |
|------|------|
| `train.json` | 训练集 |
| `dev.json` | 验证集 |

## 七、数据格式（每行JSON）
```json
{
  "text_id": "1",
  "query": "华侨村西堤1巷12栋",
  "candidate": [
    {"text": "华侨新村西堤一巷12号", "label": "部分匹配"}
  ]
}
```

## 八、输出与提交要求
- 保持与评测数据相同的**行数、顺序**
- 保留 `text_id`、`query`、`text` 原值
- 每行用 `json.dumps` 导出

## 九、评估指标
**Macro F1**

对三类标签分别计算：
- 召回率 \( R_g \)
- 精确率 \( P_g \)
- \( F1_g \)

最终：
\[
\text{Macro F1} = \frac{F1_{完全} + F1_{部分} + F1_{不匹配}}{3}
\]
